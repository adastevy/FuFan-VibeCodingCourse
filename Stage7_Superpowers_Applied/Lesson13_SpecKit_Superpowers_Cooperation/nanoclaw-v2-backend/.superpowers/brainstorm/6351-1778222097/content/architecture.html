一、核心架构：消息流转完整路径

整个系统是一个 message-broker：host 进程和 agent 容器之间没有 IPC、没有 stdin、没有文件 watcher，唯一通道是两个 SQLite 文件。

二、四个核心模块的职责边界

平台 IO 适配层

📡 src/channels/

职责：把不同平台（Discord/Slack/CLI/iMessage…）的事件统一成 InboundEvent，把 OutboundMessage 投递回平台。

  • adapter.ts — 稳定 API ChannelAdapter 接口契约
  • channel-registry.ts — 自注册中心 + 启动时初始化
  • chat-sdk-bridge.ts — 把 Chat SDK 适配器封装为 ChannelAdapter
  • cli.ts — 唯一内置适配器
  • index.ts — barrel，skill 在这里 append import

⚠️ 实际平台代码不在 trunk，住在 channels 长期分支，由 /add-discord 等 skill 拷入。

可插拔策略层

🧩 src/modules/

职责：通过 router/delivery 暴露的 hook 钩子，叠加跨切面行为。core 不依赖任何 module，缺失时 fallback 为 allow-all。

  • permissions/ — sender/access/scope 三个 gate
  • approvals/ — 通用审批原语 + OneCLI 桥
  • scheduling/ — cron 任务（pre-task script 钩子）
  • agent-to-agent/ — agent 间路由 destinations
  • self-mod/ — install_packages / add_mcp_server
  • typing/ · interactive/ · mount-security/

⚠️ modules/index.ts 加载顺序敏感：approvals 必须早于 self-mod。

持久化 + 迁移

🗃️ src/db/

职责：central DB（data/v2.db）的 schema、查询封装、迁移系统。不包含 session DB 操作（那在 session-manager.ts）。

  • connection.ts · schema.ts — 引导 + 引用 schema
  • agent-groups.ts · messaging-groups.ts · sessions.ts — 实体查询
  • dropped-messages.ts — 路由审计
  • migrations/ — 13 个 core + 3 个 module 迁移，read-only

⚠️ session DB 走 session-manager.ts + session-db.ts，开闭一次性写 是不变式。

隔离 agent runtime

📦 container/

职责：独立的 Bun 进程树。host 之外的一切：poll loop、provider 抽象、MCP 工具、formatter、destinations。

  • agent-runner/src/poll-loop.ts — 主循环（核心）
  • providers/ — Claude / mock / 可插拔 factory
  • mcp-tools/ — 7 个内置工具，barrel 自注册
  • db/ — inbound.db RO + outbound.db RW
  • destinations.ts · formatter.ts

⚠️ Bun 运行时，不是 Node。bun:sqlite 命名参数要带 $ 前缀。

四、二开建议方向

  1. 新平台接入 → 写个 ChannelAdapter，参考 cli.ts 或 chat-sdk-bridge 模式。不要 fork channels 分支，应该新建独立分支 + 自己的 install skill。
  2. 新 LLM 供应商 → 实现 AgentProvider + 在 providers/index.ts append 注册，同时在 host 端 src/providers/<name>.ts 加 container-config（参考 OpenCode）。
  3. 新策略行为（限流、内容审查、统计）→ 写 module，挂 router hook，不要改 router 本身。
  4. 给 agent 加工具 → 写 MCP tool，container 自带 server 不需要外置。
  5. 持久化新数据 → 写 module 迁移走 module-<name>-*.ts 命名，避开 core 编号区。
