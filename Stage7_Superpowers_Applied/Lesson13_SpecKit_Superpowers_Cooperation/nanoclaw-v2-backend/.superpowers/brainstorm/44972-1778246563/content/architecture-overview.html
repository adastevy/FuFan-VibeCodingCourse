4 大模块边界
📡

src/channels/

稳定 API: ChannelAdapter 接口 (adapter.ts:110)
红线: 不要改 chat-sdk-bridge.ts 核心逻辑
新增: native adapter 或 Chat SDK bridge 二选一

🧩

src/modules/

稳定 API: 6 个 register* 函数
红线: 模块禁止互相 import
加载顺序: approvals → ... → self-mod

🗄️

src/db/

三 DB: central + inbound + outbound
红线: journal_mode=DELETE 不能改 WAL
seq parity: host=even, container=odd

📦

container/

运行时: Bun（host=Node）
稳定 API: 33 文件 · provider + MCP 注册
红线: 不要改 Dockerfile/entrypoint.sh

6 个 registry 扩展点（NanoClaw 事实上的插件系统）
📡 registerChannelAdapter(name, factory)   → src/channels/channel-registry.ts
📤 registerDeliveryAction(action, handler)   → src/delivery.ts
✅ registerApprovalHandler(action, handler)   → src/modules/approvals/primitive.ts
💬 registerResponseHandler(handler)   → src/response-registry.ts
🤖 registerProvider(name, factory)   → container/agent-runner/src/providers/
🔧 registerTools([...])   → container/agent-runner/src/mcp-tools/
⚠️ 硬编码逃生口（不被 registry 覆盖）

这两处只能改 core，不能用 register 扩展

  • delivery.ts 内联 channel_type === 'agent' 路由（agent-to-agent 走的）
  • delivery.ts 内联 content.type === 'ask_question' 写 pending_questions
