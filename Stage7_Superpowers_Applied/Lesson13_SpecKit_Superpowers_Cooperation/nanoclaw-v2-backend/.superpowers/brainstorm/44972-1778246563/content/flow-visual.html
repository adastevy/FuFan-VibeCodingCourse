10用户在 Discord / Slack / Telegram 看到 agent 回复 ✨
🔥 4 条不变量（违反任何一条 = 数据丢失或死锁）
① 单写者铁律 · Host 永远不写 outbound.db，Container 永远不写 inbound.db。delivered 表写 inbound 而非 outbound 就是为了守住这条。
② journal_mode = DELETE · 不能改 WAL。WAL 的 -shm 内存映射在 Docker mount 上不刷新到 guest，容器会读到 stale snapshot 卡死。
③ Seq parity · Host=偶数 · Container=奇数。disjoint namespace 让 agent 用 seq 跨方向引用消息（"edit message #5" 不歧义）。
④ Heartbeat 是文件 mtime · 不是 DB 字段。容器 touch /workspace/.heartbeat，host sweep 60s 检查 mtime。设计动机：长跑任务（多小时 Bash）不会被误杀。
