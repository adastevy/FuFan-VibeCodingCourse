# Style · 微信公众号

## When to Use
- Writing a 公众号 long-form article or newsletter issue
- Formatting technical or industry analysis for WeChat readers
- NOT for 小红书 notes or Weibo threads

## Voice & Length
- Tone: authoritative yet accessible; define jargon on first use
- Length: 1500–3000 words
- Cadence: H2 subheading every 200–400 words

## Opening (100–200 words)
Lead with a scene or question that anchors the reader's problem before introducing the topic. Avoid "Hi everyone, today we're talking about…"

## Body Conventions
- H2 subheadings: factual and descriptive ("5月真实趋势" not "你必须知道的秘密")
- Every factual claim → cite source URL + publication name inline
- Use tables or bullet lists for comparisons and data
- One concrete example per major section

## Closing
- 2–3 sentence summary of key takeaways
- One action prompt or "推荐阅读" link
- Avoid bare boilerplate ("欢迎转发，感谢支持") as the only closing line

## Common Mistakes
| Excuse | Reality |
|--------|---------|
| 开头直接抛结论 | 读者未进入语境，很快退出 |
| 无来源引用 | 公众号读者重视可信度，裸说法掉权威性 |
| 全文无小标题 | 长文无视觉锚点，读者扫一眼就关闭 |
| 标题党 | 订阅读者打开率低，标题党反而掉粉 |
