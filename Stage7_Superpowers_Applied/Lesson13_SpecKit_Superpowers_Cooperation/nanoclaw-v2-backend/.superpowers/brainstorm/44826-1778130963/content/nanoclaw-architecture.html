    U->>CH: 平台事件
    CH->>R: onInbound()
    R->>R: resolve group → sender → agent → access gate
    R->>SM: resolve/create session
    SM->>DBin: INSERT messages_in（journal=DELETE，写完 close）
    SM->>CR: 唤醒容器
    CR->>C: docker run / 复用现有
    C->>DBin: SELECT 新消息（read-only mount）
    C->>API: Claude Agent SDK 调用
    API-->>C: 响应
    C->>DBout: INSERT messages_out
    Note over D,DBout: 出站轮询独立循环
    D->>DBout: 轮询未投递
    D->>DBin: 写 delivered 表
    D->>CH: onOutbound()
    CH->>U: 推送回平台

⚠️ 三个跨容器 SQLite 不变量（绝不能改）

  • journal_mode = DELETE — WAL 的 mmapped -shm 不会跨挂载刷新到 guest，会丢消息
  • host 写完必须 close — 长连接会冻结 container 的 page cache，让它"看不到新消息"
  • 一份 DB 单一写者 — DELETE 模式 journal-unlink 跨挂载非原子，多写会损坏
